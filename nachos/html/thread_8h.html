Detailed Description

Data structures for managing threads.

A thread represents sequential execution of code within a program. So the state of a thread includes the program counter, the processor registers, and the execution stack.

Note that because we allocate a fixed size stack for each thread, it is possible to overflow the stack – for instance, by recursing to too deep a level.
